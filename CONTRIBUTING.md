# Contributing

AIQSE is a draft proposal. It will only become useful through disagreement, application, and revision. All of the following are contributions:

- **Experience reports** — you applied any part of this on a real project. What worked, what didn't. Open an issue with the label `experience-report`. These are the most valuable contributions this project can receive.
- **Challenges** — you think a principle or dimension is wrong, redundant, or missing. Open an issue with your reasoning.
- **Worked examples** — a full spec → quality model → expected results → implementation → evidence walkthrough under `examples/`.
- **Refinements** — clearer wording, better templates, translations. Pull requests welcome.

## Ground rules

Keep discussion technical and concrete — claims backed by an example or a report beat claims backed by conviction. English is the working language; translations of the documents are welcome under `translations/`.

## Style

Documents should stay readable in minutes, not hours. Prefer tables and short YAML blocks over prose. The README must remain a 3-minute read.
